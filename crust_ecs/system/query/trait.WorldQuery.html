Expand description

A trait for querying components in a world with additional safety and utility methods.

It provides methods to produce a bitset representing the components to be queried, to get a column query for iterating over the components, and to specify the access type for the components being queried.

It’s mostly a marker trait, used to mark any query needed to be put into the Query Struct Currently, only accepts Entity and Components as well as tuple combinations of these as the T part of the trait.

When a component is used, it needs to be a reference and need to specify whether a &(const reference) is needed or a &mut (mutable reference) is needed.
